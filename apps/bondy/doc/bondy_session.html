    bondy_session (bondy v1.0.0-beta.16) View Source

    A Session (wamp session) is a transient conversation between two WAMP Peers attached to a Realm and running over a Transport.

    Bondy implementation ties the lifetime of the underlying transport connection for a WAMP connection to that of a WAMP Session i.e. establish a new transport-layer connection as part of each new session establishment.

    A Bondy Session is a not an application Session and is not a store for application specific content (an application session store should be implemented as a service i.e. a Callee).

    Currently sessions are not persistent i.e. if the connection closes the session data will be lost.
