    bondy_realm

    RealmsÂ are routing and administrative domains and act as namespaces for all resources in Bondy i.e. all users, groups, permissions, registrations and subscriptions belong to a Realm. Messages and events are routed separately for each individual realm so sessions attached to a realm wonâ€™t see message and events occurring on another realm.

    bondy_realm_wamp_api
    bondy_ref

    A bondy_ref (reference) acts as a fully qualified name for a process or callback function in a Bondy network. The reference is used by Bondy in the bondy_registry when registering procedures and subscriptions so that the bondy_router can forward and/or relay a message to a process, or callback function.

    bondy_registry

    An in-memory registry for PubSub subscriptions and Routed RPC registrations, providing pattern matching capabilities including support for WAMP's version 2.0 match policies (exact, prefix and wildcard).

    bondy_registry_entry
    An entry is a record of a RPC registration or PubSub subscription. It is stored in-memory in the Registry bondy_registry and replicated globally and is immutable.
